# AADL2 属性系统深入分析

本文档基于源码分析，深入描述 OSATE2 的属性（Property）系统架构、求值流程、工具类 API 和模态值处理机制。

源码核心位置：`core/org.osate.aadl2`、`core/org.osate.xtext.aadl2.properties`

---

## 1. 属性系统架构

属性系统分四层：

```
元模型层（Ecore）
  aadl2.ecore 中定义 Property、PropertyType、PropertyExpression、
  PropertyAssociation、ModalPropertyValue 等核心 EClass
       ↓
接口层（Java Interfaces）
  org.osate.aadl2 包中的 *.java 接口文件
       ↓
实现层（Java Implementations）
  org.osate.aadl2.impl 包，核心：NamedElementImpl（属性查询逻辑）
       ↓
工具类层
  PropertyUtils     — 通用属性获取（含异常处理）
  GetProperties     — 特定属性集的强类型 getter
  PropertyAcc       — 属性值累积器（内部查询机制）
```

---

## 2. PropertyType 层次

```
PropertyType (abstract, extends Type)
├── NonListType (abstract)
│   ├── NumberType (abstract)
│   │   ├── IntegerType
│   │   └── RealType
│   ├── UnitsType            — 单位定义（ms, s, MIPS, KB 等）
│   ├── StringType
│   ├── BooleanType
│   ├── EnumerationType      — 枚举（如调度协议 Periodic/Sporadic）
│   ├── RangeType            — 范围（最小值、最大值、增量）
│   ├── RecordType           — 记录（命名字段集合）
│   └── ClassifierType       — 对分类器的引用
└── ListType
    └── elementType → PropertyType  — 元素类型（支持嵌套列表）
```

---

## 3. PropertyExpression 层次

```
PropertyExpression (abstract)
├── PropertyValue (abstract)
│   ├── IntegerLiteral         — 整数值（含可选单位）
│   ├── RealLiteral            — 浮点值（含可选单位）
│   ├── StringLiteral          — 字符串值
│   ├── BooleanLiteral         — 布尔值
│   ├── EnumerationLiteral     — 枚举字面值
│   ├── RangeValue             — 范围值
│   │   ├── minimum: PropertyExpression
│   │   ├── maximum: PropertyExpression
│   │   └── delta:   PropertyExpression (可选)
│   ├── RecordValue            — 记录值
│   │   └── ownedFieldValues: BasicPropertyAssociation[]
│   ├── ListValue              — 列表值
│   │   └── ownedListElements: PropertyExpression[]
│   ├── ClassifierValue        — 分类器引用
│   ├── ReferenceValue         — 实例对象引用
│   ├── ComputedValue          — 计算值（函数调用）
│   └── NamedValue             — 引用属性常量或其他属性
└── ModalPropertyValue         — 模态包装器
    ├── ownedValue: PropertyExpression  — 实际值
    └── inModes: Mode[]                 — 适用模式（空 = 所有模式默认）
```

## 4. Property 与 PropertyAssociation

### 4.1 Property 接口

```java
public interface Property extends BasicProperty, AbstractNamedValue {
    boolean isInherit();              // 是否继承到子组件
    PropertyExpression getDefaultValue();   // 属性的默认值
    EList<PropertyOwner> getAppliesTos();   // 适用的元素类型
    boolean isEmptyListDefault();
}

public interface BasicProperty extends TypedElement {
    PropertyType getPropertyType();   // 属性的类型（owned 或 referenced）
    boolean isList();                 // 是否为列表属性
}
```

**作用域规则**：
- `getAppliesTos()` 决定属性可以附加到哪类元素（通过 MetaclassReference 或 ClassifierValue）
- `isInherit() == true`：属性从组件类型继承到组件实现

### 4.2 PropertyAssociation 结构

```java
public interface PropertyAssociation extends Element {
    Property getProperty();                    // 关联的属性定义
    EList<ContainedNamedElement> getAppliesTos(); // applies to 子句（空 = 容器本身）
    EList<Classifier> getInBindings();         // in binding 子句
    boolean isAppend();                        // 是否追加到列表
    boolean isConstant();                      // 值是否常量（不可覆盖）
    EList<ModalPropertyValue> getOwnedValues(); // 实际值列表（含模态包装）
}
```

**applies to 子句**：
- `getAppliesTos()` 为空：关联附加到声明它的元素本身
- 非空：关联附加到容器中的指定子元素（如 `applies to port1.data_port`）
- `ContainedNamedElement` 通过 `ContainmentPathElement` 链支持多级路径

## 5. 属性求值流程

### 5.1 调用入口

```java
// 使用工具类（推荐）
PropertyExpression pe = PropertyUtils.getSimplePropertyValue(element, property);
long value = PropertyUtils.getIntegerValue(element, property, defaultVal);
double time = PropertyUtils.getScaledNumberValue(element, property, TimeUnits.MS);

// 直接调用（低层）
PropertyAcc acc = element.getPropertyValue(property, false);
PropertyAssociation pa = acc.first();
```

### 5.2 NamedElementImpl 查询逻辑

```java
// 核心实现（NamedElementImpl）

// 1. 适用性检查
boolean acceptsProperty(Property property) {
    for (PropertyOwner appliesTo : property.getAppliesTos()) {
        if (appliesTo instanceof MetaclassReference) {
            if (metaRef.getMetaclass().isSuperTypeOf(eClass())) return true;
        }
        if (appliesTo instanceof ClassifierValue) return true;
    }
    return false;
}

// 2. 循环检测（ThreadLocal 栈）
private static final ThreadLocal<LinkedList<Property>> lookupStack =
    ThreadLocal.withInitial(LinkedList::new);

// 3. PropertyAcc 搜索
void getPropertyValueInternal(Property pn, PropertyAcc pas, ...) {
    // 先搜索 contains 关联（applies to 指向此元素的子句）
    pas.addLocalContained(this, getContainingClassifier());
    // 再搜索本地关联（applies to 为空）
    pas.addLocal(this);
}
```

### 5.3 PropertyAcc（属性累积器）

```java
public class PropertyAcc {
    // add*() 返回 true = 找到非列表值，停止搜索
    // add*() 返回 false = 继续搜索（列表属性需要收集所有值）
    public boolean addLocal(NamedElement target);
    public boolean addLocalContained(NamedElement target, NamedElement container);
    public PropertyAssociation first();   // 获取最高优先级的关联
    public List<PropertyAssociation> getAssociations(); // 所有收集到的关联
}
```

### 5.4 完整搜索顺序

```
element.getNonModalPropertyValue(property)
    ↓
1. getPropertyValue(property, false)
   └─> getPropertyValueInternal()
       ├─ 搜索当前元素自身的 PropertyAssociation（本地）
       └─ 搜索容器的 applies-to 关联（contained）

2. 如果未找到：
   └─ 使用 property.getDefaultValue()

3. 如果是模态属性（InstanceObject 上）：
   └─ 按当前 SOM 选择 ModalPropertyValue（见第 6 节）
```

## 6. 模态属性值处理

### 6.1 ModalPropertyValue 求值

```java
// getNonModalPropertyValue() 中对模态属性的处理
PropertyAssociation pa = getPropertyValue(property, false).first();

if (pa.isModal()) {
    SystemOperationMode som = systemInstance.getCurrentSystemOperationMode();
    PropertyExpression defaultPE = null;

    for (ModalPropertyValue mpv : pa.getOwnedValues()) {
        if (mpv.getInModes() == null || mpv.getInModes().isEmpty()) {
            defaultPE = mpv.getOwnedValue();       // 保存默认值
        } else if (mpv.getInModes().contains(som)) {
            return mpv.getOwnedValue();             // 精确匹配当前模式
        }
    }
    return (defaultPE != null) ? defaultPE : property.getDefaultValue();
}
```

### 6.2 InModes 语义

| InModes | 含义 |
|---------|------|
| 空（null 或 empty） | 默认值，适用于所有未匹配模式 |
| 含 Mode m | 仅当系统处于模式 m 时生效 |
| 含多个 Mode | 所列模式均适用 |

### 6.3 SOM 迭代分析模式

分析多个系统操作模式时的标准模式：

```java
for (SystemOperationMode som : systemInstance.getSystemOperationModes()) {
    systemInstance.setCurrentSystemOperationMode(som);
    // 此时 getNonModalPropertyValue() 自动选择对应 SOM 的值
    double period = PropertyUtils.getScaledNumberValue(thread, periodProp, TimeUnits.MS);
    // ... 分析逻辑
    systemInstance.clearCurrentSystemOperationMode();
}
```

### 7.4 异常类型

| 异常 | 含义 |
|------|------|
| `PropertyNotPresentException` | 属性未设置（无关联且无默认值） |
| `PropertyIsModalException` | 对模态属性使用非模态查询（需先设置 SOM） |
| `PropertyIsListException` | 对列表属性使用标量查询 |
| `PropertyDoesNotApplyToHolderException` | 属性不适用于此元素类型 |
| `InvalidModelException` | 模型不一致 |
| `IllegalStateException` | 循环属性引用 |

### 8.3 分析插件中的标准用法

```java
// 典型分析 Handler 中的属性初始化
@Override
protected void initPropertyReferences() {
    // 方式 1：通过 lookupPropertyDefinition
    periodProperty  = GetProperties.lookupPropertyDefinition(
        resource, "Timing_Properties", "Period");
    latencyProperty = GetProperties.lookupPropertyDefinition(
        resource, "Timing_Properties", "Latency");

    // 方式 2：通过 AbstractAaxlHandler.lookupPropertyDefinition（同等效果）
    deadlineProperty = lookupPropertyDefinition("Timing_Properties", "Deadline");
}

// 分析执行时使用
@Override
protected void analyzeInstanceModel(..., SystemInstance root, ...) {
    for (ComponentInstance thread : root.getAllComponentInstances()) {
        double period = GetProperties.getPeriodinMS(thread);
        double mips   = GetProperties.getMIPSBudgetInMIPS(thread, 0.0);
        List<ComponentInstance> processors =
            GetProperties.getActualProcessorBinding(thread);
    }
}
```

## 9. PropertySet 作用域

```java
public interface PropertySet extends Namespace, ModelUnit {
    EList<PropertyType>     getOwnedPropertyTypes();
    EList<Property>         getOwnedProperties();
    EList<PropertyConstant> getOwnedPropertyConstants();
    EList<AadlPackage>      getImportedUnits();  // 导入的其他单位
}
```

**可见性规则**：
1. 本地作用域：PropertySet 内定义的 Property/Type/Constant
2. 导入作用域：通过 `importedUnits` 显式导入的其他 PropertySet
3. 全局查找：`Aadl2GlobalScopeUtil.get(context, Property, qualifiedName)` 可跨文件查找
4. 限定名称格式：`PropertySetName::PropertyName`（全局唯一）

## 10. 属性使用完整示例

### 10.1 查询整数属性

```java
// 带默认值（安全）
long stackSize = PropertyUtils.getIntegerValue(thread, stackSizeProp, 4096L);

// 原始表达式（需类型转换）
PropertyExpression pe = PropertyUtils.getSimplePropertyValue(thread, stackSizeProp);
if (pe instanceof IntegerLiteral) {
    long val = ((IntegerLiteral) pe).getValue();
    UnitLiteral unit = ((IntegerLiteral) pe).getUnit();
}
```

### 10.2 查询带单位的实数

```java
UnitLiteral ms = GetProperties.findUnitLiteral(periodProp, "ms");
double period = PropertyUtils.getScaledNumberValue(thread, periodProp, ms, 10.0);
// 自动将属性值（无论原单位是 ms/s/us）换算为毫秒
```

### 10.3 查询列表属性

```java
// 获取绑定处理器列表
List<ComponentInstance> processors =
    GetProperties.getActualProcessorBinding(threadInstance);

// 获取字符串列表
PropertyExpression pe = PropertyUtils.getSimplePropertyListValue(comp, sourceTextProp);
if (pe instanceof ListValue) {
    for (PropertyExpression elem : ((ListValue) pe).getOwnedListElements()) {
        String text = ((StringLiteral) elem).getValue();
    }
}
```

### 10.4 查询枚举属性

```java
EnumerationLiteral protocol = PropertyUtils.getEnumLiteral(thread, dispatchProtocolProp);
if ("Periodic".equals(protocol.getName())) {
    // 周期性线程处理
}
```

### 10.5 模态属性查询

```java
// 必须先设置 SOM
systemInstance.setCurrentSystemOperationMode(som);
try {
    // 此时 getNonModalPropertyValue() 自动选择对应 SOM 的属性值
    double period = GetProperties.getPeriodinMS(thread);
} finally {
    systemInstance.clearCurrentSystemOperationMode();
}
```

## 11. 自定义属性集开发

### 11.1 定义属性集（.aadl 文件）

```aadl
property set MyProject_Properties is
  with Data_Model, SEI;

  Max_Retry_Count: aadlinteger 0 .. 10 => 3
    applies to (thread, process);

  Message_Size: SEI::Bytes_Range => 0 B .. 1024 B
    applies to (port);

  Is_Safety_Critical: aadlboolean => false
    applies to (system, process, thread, device);
end MyProject_Properties;
```
