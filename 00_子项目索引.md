# OSATE2 子项目索引

本文档提供了OSATE2项目所有子项目的概览和导航。

## 项目信息
- **项目名称**: OSATE2 (Open Source AADL Tool Environment)
- **当前版本**: 2.18.0-SNAPSHOT
- **许可证**: Eclipse Public License 2.0
- **组织**: Carnegie Mellon University Software Engineering Institute

## 子项目结构

OSATE2项目按功能模块组织，共包含10个主要子项目：

### 1. Core - 核心模块 ⭐
**路径**: `osate2/core/`
**文档**: [core/README.md](./core/README.md)

**核心功能**:
- AADL2元模型实现
- 语言解析和验证
- 模型实例化引擎
- 基础UI框架

**关键组件**:
- org.osate.aadl2 - AADL核心API
- org.osate.xtext.aadl2 - 语法解析
- org.osate.aadl2.instantiation - 实例化引擎

**重要性**: ⭐⭐⭐⭐⭐ (所有其他模块的基础)

---

### 2. Analyses - 分析模块 ⭐
**路径**: `osate2/analyses/`
**文档**: [analyses/README.md](./analyses/README.md)

**核心功能**:
- 架构分析
- 流延迟分析
- 资源预算分析
- 模型导入工具

**关键组件**:
- org.osate.analysis.flows - 流分析
- org.osate.analysis.resource.budgets - 资源分析
- org.osate.importer - 模型导入

**应用领域**: 实时系统、嵌入式系统、性能验证

**重要性**: ⭐⭐⭐⭐⭐ (核心价值所在)

---

### 3. EMV2 - 错误模型版本2 ⭐
**路径**: `osate2/emv2/`
**文档**: [emv2/README.md](./emv2/README.md)

**核心功能**:
- 错误建模
- 故障树分析(FTA)
- 错误传播分析
- 安全性分析

**关键组件**:
- org.osate.xtext.aadl2.errormodel - 错误模型语言
- org.osate.aadl2.errormodel.faulttree - 故障树
- org.osate.aadl2.errormodel.analysis - 分析工具

**标准支持**: ARP4761, DO-178C, ISO 26262, IEC 61508

**重要性**: ⭐⭐⭐⭐⭐ (安全关键系统必备)

---

### 4. ALISA - 架构主导的增量系统保证 ⭐
**路径**: `osate2/alisa/`
**文档**: [alisa/README.md](./alisa/README.md)

**核心功能**:
- 需求规范(ReqSpec)
- 验证计划(Verify)
- 保证案例(Assure)
- 需求追踪

**关键组件**:
- org.osate.reqspec - 需求定义
- org.osate.verify - 验证方法
- org.osate.assure - 保证执行

**工作流**: 需求 → 验证 → 证据 → 报告

**重要性**: ⭐⭐⭐⭐ (认证和合规必备)

---

## 技术栈总览

### 基础框架
- **Eclipse Platform** - RCP应用框架
- **EMF** (Eclipse Modeling Framework) - 元模型和模型处理
- **Xtext** - DSL开发框架
- **Tycho** - Eclipse插件构建

### 语言和工具
- **Java** - 主要编程语言
- **Xtend** - 模板和生成
- **Maven** - 项目管理
- **Git** - 版本控制

### 图形技术
- **JavaFX** - 图形界面
- **e(fx)clipse** - Eclipse和JavaFX集成

### 测试和质量
- **JUnit** - 单元测试
- **SWTBot** - UI测试
- **JaCoCo** - 代码覆盖率
- **Checkstyle** - 代码风格

## 学习路径建议

### 入门路径 (1-2周)
1. 阅读项目概述文档
2. 安装开发环境 (Setup)
3. 学习AADL基础 (Examples基础示例)
4. 了解Core模块核心概念
5. 使用GE创建简单模型

### 进阶路径 (1-2个月)
1. 深入Core模块API
2. 学习Analyses分析功能
3. 掌握EMV2错误建模
4. 理解ALISA验证框架
5. 学习BA行为建模

### 专家路径 (持续学习)
1. 研究各模块源代码
2. 开发自定义分析插件
3. 扩展工具功能
4. 贡献代码和示例
5. 深入研究AADL标准
